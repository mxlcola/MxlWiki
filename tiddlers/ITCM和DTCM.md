### ITCM 和 DTCM

**TCM = Tightly Coupled Memory**（紧密耦合存储器）

- **ITCM**：Instruction TCM，存**指令**（代码）
- **DTCM**：Data TCM，存**数据**（变量）

------

### 它有什么特别的

TCM 是**直连 CPU 内核的专属 RAM**，不经过总线、不经过 Cache，零等待访问：



```
普通 RAM 访问路径：
  CPU → Cache → 总线矩阵 → AXI SRAM     （可能有等待周期）

TCM 访问路径：
  CPU → TCM                               （零等待，最快）
```

------

### H743 的内存布局

| 内存       | 地址         | 大小  | 特点                   |
| ---------- | ------------ | ----- | ---------------------- |
| **ITCM**   | `0x00000000` | 64KB  | CPU 专属，跑代码最快   |
| **DTCM**   | `0x20000000` | 128KB | CPU 专属，读写数据最快 |
| AXI SRAM   | `0x24000000` | 512KB | 大容量，走 Cache       |
| D2 SRAM1-3 | `0x30000000` | 288KB | DMA 可访问             |
| D3 SRAM4   | `0x38000000` | 64KB  | 你放 DMA 缓冲区的地方  |

------

### 关键特性

**优点**：

- 速度最快，零等待周期，确定性延迟
- 不受 Cache 命中/未命中的影响，时序稳定

**限制**：

- **DMA 不能访问 TCM**——因为 TCM 直连 CPU，不挂在总线上，DMA 控制器看不到这块内存
- 容量小——ITCM 64KB，DTCM 128KB

------

### 典型用法



```
ITCM（64KB）→ 放对实时性要求极高的代码
              比如：中断处理函数、DSP 算法、PID 控制循环

DTCM（128KB）→ 放频繁读写的关键数据
              比如：栈空间、DSP 计算的临时数组、查找表
```

------

### 跟你项目的关系

你项目里的音频 DSP 计算（RMS、倍频程分析）如果对实时性要求很高，理论上可以：

- 把 DSP 核心函数放到 **ITCM** 执行
- 把计算用的临时数组放到 **DTCM**

但 **DMA 缓冲区绝对不能放 TCM**，因为 DMA 访问不到。这也是你把 DMA 缓冲区放到 SRAM4 而不是 DTCM 的原因之一。

------

### 一句话总结

TCM 就是 **CPU 的"私有高速内存"**，速度最快但 DMA 用不了。ITCM 放代码，DTCM 放数据。
