不是，**只有部分系列有**。

------

### 简单分类

| 内核       | 代表型号                  | 有没有 MPU                     |
| ---------- | ------------------------- | ------------------------------ |
| Cortex-M0  | STM32F0、STM32L0          | **没有**                       |
| Cortex-M0+ | STM32G0、STM32L0+         | **部分有**（可选）             |
| Cortex-M3  | STM32F1、STM32F2、STM32L1 | **有**（8个Region）            |
| Cortex-M4  | STM32F4、STM32G4、STM32L4 | **有**（8个Region）            |
| Cortex-M7  | **STM32F7、STM32H7**      | **有**（16个Region）           |
| Cortex-M33 | STM32L5、STM32U5、STM32H5 | **有**（增强版，支持安全区域） |

------

### 关键点

- **M0 系列没有 MPU**，内核太精简了，成本和功耗优先
- **M3/M4 有 MPU 但很少用**，因为没有 Cache，不存在一致性问题，MPU 主要用于 RTOS 的内存保护（防止任务越界访问）
- **M7 上 MPU 几乎必须配置**，因为有 D-Cache，不配 MPU 的话所有内存默认策略可能导致 DMA 数据异常

所以 MPU 是 ARM 内核提供的可选功能，不是 STM32 特有的。**凡是用 Cortex-M3 及以上内核的 MCU（不限于 STM32）基本都有 MPU**，但低端的 M0 系列为了控制成本和面积就省掉了。
